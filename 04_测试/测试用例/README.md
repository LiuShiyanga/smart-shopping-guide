## 测试用例模板

每个测试用例文件使用以下模板格式：

```markdown
# [测试用例ID] 测试用例名称

## 测试目标
描述本测试用例的目标和要测试的功能点

## 前置条件
列出执行测试前需要满足的条件

## 测试数据
列出测试所需的数据

## 测试步骤
1. 步骤1
2. 步骤2
3. 步骤3
...

## 预期结果
描述每个步骤预期的结果

## 实际结果
[测试执行时填写]

## 测试结论
[ ] 通过
[ ] 失败

## 备注
其他相关说明
```

## 用例编写指南

### 1. 用例覆盖范围

测试用例应覆盖以下方面：
- 正常场景（Happy Path）
- 边界条件
- 异常情况
- 用户体验相关点

### 2. 用例命名规范

- 用例ID遵循"[模块]-[功能]-[序号]"格式
- 用例名称简洁明了，反映测试内容

### 3. 用例优先级

用例按优先级分类：
- P0: 核心功能，必须测试
- P1: 重要功能，应该测试
- P2: 次要功能，如时间允许测试
- P3: 低优先级功能，可选测试

## 用例维护

测试用例应随功能变更及时更新。用例更新记录在对应文件的版本历史部分，包含：
- 更新日期
- 更新人
- 更新内容
- 更新原因
